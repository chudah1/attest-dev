# Issue a root credential for your orchestrator
const { token } = await client.issue({
  agentId:     "orchestrator-v1",
  userId:      "usr_alice",
  scope:       ["research:read", "gmail:send"],
  instruction: "summarise inbox and send digest",
});

# Delegate a narrower credential to a sub-agent
const { token: childToken } = await client.delegate({
  parentToken: token,
  childAgent:  "email-agent-v2",
  childScope:  ["gmail:send"],  // must be ⊆ parent scope
});

# Revoke the entire chain instantly
await client.revoke(token);
⚡

Intent binding

The original instruction is hashed once and carried through the full delegation chain. Months later, you can recompute the hash and prove the action still maps back to the human request.

🔌

MCP tool protection

Drop withAttest() onto an MCP server and every tool call is credential-gated automatically. The server stays your source of truth for required scopes.

🎯

Scoped delegation

Child agents only receive scopes that are a strict subset of the parent. Scope never expands as the task fan-outs across sub-agents.

🧵

Cascade revocation

Revoke a root credential once and the entire task tree becomes invalid immediately. No descendant handoff stays live after the parent is cut off.

📋

Audit evidence

Every issue, delegation, and revocation lands in a tamper-evident audit log so you can produce evidence instead of just logs.

🔐

Signed credentials

RS256-signed JWTs make verification offline and forgery-resistant, so the proof travels with the token and not just the backend.

How it works

From instruction to proof

Attest turns a human request into a delegated authority chain that can be enforced in real time and verified later.

1

User authorizes a task

Your backend calls POST /v1/credentials with the agent ID, user ID, permitted scopes, and original instruction. Attest returns a signed root credential bound to that intent.

2

Orchestrator delegates to sub-agents

The orchestrator calls POST /v1/credentials/delegate, narrowing scope for each child agent. Every child token carries the full ancestry chain.

3

MCP tools verify on every call

withAttest() intercepts every tool invocation, verifies the JWT against your org's JWKS, and checks the required scope before your handler runs.

4

Revoke any time

Call DELETE /v1/credentials/:jti to revoke a token. All descendants are invalidated immediately through cascade revocation.
