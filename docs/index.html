Cryptographic credentials for AI agents.
Every agent in your pipeline gets a signed, scoped, delegatable credential. Know exactly who authorized what — and revoke it instantly.

Get started View on GitHub Live demo → 
# Issue a root credential for your orchestrator
const { token } = await client.issue({
  agentId:     "orchestrator-v1",
  userId:      "usr_alice",
  scope:       ["research:read", "gmail:send"],
  instruction: "summarise inbox and send digest",
});

# Delegate a narrower credential to a sub-agent
const { token: childToken } = await client.delegate({
  parentToken: token,
  childAgent:  "email-agent-v2",
  childScope:  ["gmail:send"],  // must be ⊆ parent scope
});

# Revoke the entire chain instantly
await client.revoke(token);
⚡

Intent locking

The original instruction is SHA-256 hashed and baked into every credential in the chain. Six months later, a regulator asks what authorised that agent action. You recompute the hash. It matches. Proof.

🔐

RS256-signed JWTs

Every credential is cryptographically signed with your org's RSA-2048 key. Forgery is impossible, verification is instant.

🎯

Scoped delegation

Child agents can only receive scopes that are a strict subset of their parent's. Privilege escalation is blocked at issuance time.

🧵

Full chain of custody

Every credential carries its ancestry chain and a depth counter. Revoking a parent cascades to all descendants automatically.

📋

Audit log

Every issue, delegation, and revocation is appended to a tamper-evident audit log queryable by task ID.

🔌

MCP middleware

Drop withAttest() onto any MCP server. Tools enforce credentials automatically — no manual wiring needed.

How it works

From user intent to auditable action

Attest implements a delegation chain: each agent can only act within the bounds set by the agent above it.

1

User authorizes a task

Your backend calls POST /v1/credentials with the agent ID, user ID, permitted scopes, and the original instruction. Attest returns a signed root token.

2

Orchestrator delegates to sub-agents

The orchestrator calls POST /v1/credentials/delegate, narrowing the scope for each child agent. Each child token carries the full ancestry chain.

3

MCP tools verify on every call

withAttest() intercepts every tool invocation, verifies the JWT against your org's JWKS, and checks that the required scope is granted — before your handler runs.

4

Revoke any time

Call DELETE /v1/credentials/:jti to revoke a token. All descendants are invalidated immediately via cascade revocation.
