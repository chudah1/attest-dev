MCP Quickstart

Protect one MCP tool. See the result immediately.

This is the shortest real path into Attest: wrap an existing MCP server with withAttest(), issue only the scope that tool needs, watch one valid call go through, then block one that should never have been allowed.

1

Install the SDK

npm install @attest-dev/sdk @modelcontextprotocol/sdk
2

Wrap your MCP server

Add withAttest() to your existing MCP server. This intercepts every tool call and checks the caller's credential scope.

import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { withAttest } from "@attest-dev/sdk/mcp";

const server = new McpServer({ name: "my-tools", version: "1.0.0" });
const protectedServer = withAttest(server, {
  issuerUri: "https://api.attestdev.com",
});

protectedServer.tool("send_email", schema, handler);
3

Run the protected call path

Issue a credential, make a tool call, and see it succeed or get blocked based on scope.

# Create an org and keep the API key shown once
curl -X POST https://api.attestdev.com/v1/orgs \
  -H "Content-Type: application/json" \
  -d '{"name":"acme-corp"}'

# Issue only the discovered scope for send_email
curl -X POST https://api.attestdev.com/v1/credentials \
  -H "Authorization: Bearer att_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "orchestrator-v1",
    "user_id": "usr_alice",
    "scope": ["email:send"],
    "instruction": "Send the weekly digest"
  }'
4

See what happened

Open the dashboard to inspect the task tree, or export the evidence packet and verify it independently.

Scoped credentials

Each agent gets a JWT with only the permissions it needs. Authority narrows at each delegation level.

Audit trail

Every issuance, delegation, tool call, and revocation is recorded in a hash-chained append-only log.

Evidence packet

Export a signed JSON packet containing the full event chain. Verify it independently in your browser.
