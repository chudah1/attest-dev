Public verification

Drop in a packet. Verify it on site.

This page checks the canonical packet hash, RS256 signature, and append-only audit chain in your browser. If the packet includes org metadata, Attest can load the JWKS automatically.

No dashboard required Web Crypto in-browser Works with exported JSON Optional custom JWKS
What this checks
Independent verification, not server trust.

The verifier recomputes the packet hash locally, selects the signing key by kid, validates the signature, and checks the audit chain linkage in the event stream.

Canonical packet hash Detects any mutation to the signed export artifact.
JWKS-backed signature Uses the org public key set and the packet signature_kid.
Append-only audit chain Checks that each event links to the previous event hash.
